# Goal Team Map

The backend registry defines nine classes: Implementation; Debugging; Test and
Verification; Architecture and Code Review; Research and Experiment;
Documentation and Communication; Release and Reliability; Security and Risk;
Frontend and UI Engineering.

Start with one primary expert. Code, experiment, release, and security claims
also require an eligible verifier instance that did not execute the governed
unit. Low-risk read-only preparation may use a self-check.
