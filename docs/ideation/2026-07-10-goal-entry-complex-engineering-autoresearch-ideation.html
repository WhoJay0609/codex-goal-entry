Evidence before direction

Grounding Context

Codebase Context. 独立仓库只发布轻量 goal-entry：一个 Python stdlib resolver、入口合同、架构说明和 quick validator。仓库规则明确要求它保持 router-only，详细执行协议继续由外部 goal-* child skills 所有。

已安装环境远比公开包丰富：它已经拥有 preflight、context、objective、dispatch/team、backend、trace/closeout，以及 codex-autoresearch、迭代状态、metric trust、benchmark weakening guard、resume summary 和研究/工程/bridge preset。真正的缺口是入口无法把请求可靠地映射到这些能力。

现场回放暴露了两个直接问题：显式“跨模块 + 并行 + 回归 + 发布”的工程请求能进入 full_autonomous；但自然表达的“开展自动科研、检索文献、提出假设、运行消融、分析结果”会因 分析 命中 review 而落入 report_only。同样，“继续昨天失败的 trial 并重跑”在没有外部 active-goal snapshot 时也会回落到 report_only。

还有一个当前态漂移：standalone SKILL.md 仍声明 harness-agent-for-goal 与 harness-agents-md-context “remain installed”，但 live skill tree 已不存在这些目录。这说明 public package 与 full stack 的契约一致性不能继续只靠文字维护。

Boundary that must survive

不要把科研或工程执行细节重新塞回 goal-entry。 它应只负责产生更有语义、可验证、可协商的路由结果；阶段图、实验合同、团队和证据门继续由 child owners 承接。

外部研究工具连续两次在网络层失败，因此本轮排序不使用任何外部主张；结论主要来自当前仓库、installed skill stack、backend scripts、autoresearch contract 和历史经验。独立 verifier dispatch 也受当前 harness 映射限制，本轮由主线程逐条核验所有 load-bearing basis。

A1. Goal Runtime Profile packs

把已有能力连接成可选的运行画像，而不是继续扩张入口正文。

Description
定义轻量 declarative packs，把 complex_engineering 与 scientific_autoresearch 映射到生命周期阶段、capability requirements、默认 team bias、artifact contract、evidence gate 与语义 closeout。goal-entry 只选择 profile，child skills/backend 继续拥有执行协议。
Axis
Team and skill routing
Basis
direct: references/architecture.md:3-22 已将职责拆给 child owners；team_presets.py:81-157 已有 research、bridge、engineering、delivery 和 evidence 队形。

这是连接现有碎片化能力的最小架构，同时完整保留 public router-only 边界。
Downsides
需要一个小型、版本化 profile schema，并明确 capability negotiation 与缺失 provider 的降级行为。

A2. Typed three-stage resolver

先解析“想做什么”，再判断“属于什么工作”，最后识别“现在处于哪里”。

Description
将一次混合 regex pass 拆成三个独立、确定性、可测试阶段：intent（report/plan/execute）、task profile（engineering/research/bridge/general）与 lifecycle state（new/resume/recover/close）。输出版本化 typed result、stage evidence、ambiguities 与 v1 compatibility projection。
Axis
Intent and policy routing
Basis
direct: resolve_goal_entry.py:203-262 把 mode、tier 与通用软件 route 混在一起；:371-389 缺少 profile、lifecycle 和 state 字段。现场请求证明 分析 可把科研执行误判为报告。
Rationale
把错误从“规则越来越多”提升为“语义维度缺失”，同时仍保持 deterministic policy。
Downsides
必须管理 schema migration、规则 precedence 和中文/英文等价性；不应引入不可解释的自由模型分类作为唯一真相。
Confidence
92%

A3. Contract-Conformance Evaluation Harness

先把目标行为固定成可执行合同，再升级 resolver。

Description
建立真实中英文 golden journeys，覆盖复杂工程、research bootstrap、实验迭代、paper-to-code、失败恢复、no-execution 与 closeout；再加 metamorphic invariants 和 standalone/full-stack differential conformance，断言完整 typed decision。
Axis
Verification and portability
Basis
direct: scripts/quick_validate.py:68-115 只有一个英文 “PLEASE IMPLEMENT THIS PLAN” 的 in-process assertion 与等价 CLI smoke；SKILL.md:16-25 声明的 legacy entries 已与 live installed tree 不一致。
Rationale
这是最安全的第一落地点，能在架构演进前先捕获当前误路由，并防止后续回到 regex whack-a-mole。
Downsides
场景预期本身就是 policy，需要 owner 审核；过度快照整个 JSON 会造成不必要的脆弱性。

A4. Evidence-Carrying Resume Cursor

让“继续做”绑定可验证状态，而不是绑定聊天记忆或一句关键词。

Description
发现 active Goal、harness run、research-state.yaml 与 checkpoint artifacts，选择最后一个已验证 checkpoint，并返回 state source、active/blocked work、next gate、conflicts 与 confidence。找不到状态时进入 targeted handshake，不再静默回落为 report_only。
Axis
Orchestration graph and recovery
Basis
direct: resolve_goal_entry.py:221-228 只有外部传入 active-goal 才能 bind；summarize_goal_resume.py:70-130 已能从 queue、worktree、cleanup 与 goal-sync 重建下一动作。
Rationale
跨 session 中断与恢复是复杂工程和 autoresearch 的共同主故障面，现有 backend 已提供大半证据。
Downsides
多个 state format 之间需要严格 precedence/conflict rules；artifact-derived inference 必须显式低置信，不能覆盖声明状态。

A5. Complex Engineering Phase Graph

把跨模块交付编译为可恢复、可并行、可集成的 dependency graph。

Description
为 complex-engineering profile 形成 context → architecture decision → implementation branches → integration → verification → release/closeout 的阶段 DAG。每个节点生成 integration-ready delegation packet：owner、scope、dependencies、wait rules、output contract、acceptance mapping、conflict owner 与 fallback。
Axis
Orchestration graph and recovery
Basis
direct: goal-dispatch/SKILL.md:23-39 已要求 context、dependency、wait、output、integration、conflict、fallback 和 non-overlapping writes，但 entry 只给出一个 generic route。
Rationale
将现有强 prose contract 变成可审计执行图，直接改善多模块、多 agent 与 worktree 合流。
Downsides
小项目不应被强制建图；activation 必须受 profile/horizon 控制，避免流程税。

A6. Scientific Two-Loop Experiment Contract

把科研方法变成 durable Goal semantics，而不是把科研当作“更长的软件任务”。

Description
显式表示 bootstrap、inner experiments、outer synthesis 与 finalize；要求 protocol-before-results、confirmatory/exploratory label、baseline/metric trust、sanity checks、budget epochs，以及 DEEPEN/BROADEN/PIVOT/CONCLUDE 转移。
Axis
Scientific lifecycle and evidence
Basis
direct: autoresearch SKILL.md:60-148 定义循环与实验纪律，:188-227 定义方向转移；init_auto_harness.py:515-645 已保存 iteration、stopping、benchmark 与 metric-trust state。
Rationale
本地已有研究 workflow 与 backend primitives，只缺少 Goal 控制面的正式接线与恢复语义。
Downsides
必须防止与现有 autoresearch/experiment skills 形成第二个 scheduler；profile 应引用它们而非复制执行步骤。

A7. Scientific Claim Firewall

实验“跑完”不等于 claim 可以写进论文。

Description
在 experiment completion 与 manuscript mutation 之间设置 hard gate：experiment integrity、result-to-claim verdict、limitations 和 independent evidence review 必须落盘。partial/no 只能缩小 claim、保留负结果或安排补充实验。
Axis
Scientific lifecycle and evidence
Basis
direct: experiment-audit/SKILL.md:13-27 列出 fake ground truth、self-normalization、phantom result 与 scope inflation；result-to-claim/SKILL.md:15-20,92-138,180-187 已按 integrity 与 yes/partial/no verdict 路由。
Rationale
它直接保护 autoresearch 最危险的输出：语气坚定但证据不足的科学结论。
Downsides
现有 experiment-audit 是 advisory；新 profile 必须明确哪些失败真正阻断 paper write，避免把所有 warn 都升级为永久阻塞。
