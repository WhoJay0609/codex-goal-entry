Evidence before direction

Grounding Context

Codebase Context. 独立仓库只发布轻量 goal-entry：一个 Python stdlib resolver、入口合同、架构说明和 quick validator。仓库规则明确要求它保持 router-only，详细执行协议继续由外部 goal-* child skills 所有。

已安装环境远比公开包丰富：它已经拥有 preflight、context、objective、dispatch/team、backend、trace/closeout，以及 codex-autoresearch、迭代状态、metric trust、benchmark weakening guard、resume summary 和研究/工程/bridge preset。真正的缺口是入口无法把请求可靠地映射到这些能力。

现场回放暴露了两个直接问题：显式“跨模块 + 并行 + 回归 + 发布”的工程请求能进入 full_autonomous；但自然表达的“开展自动科研、检索文献、提出假设、运行消融、分析结果”会因 分析 命中 review 而落入 report_only。同样，“继续昨天失败的 trial 并重跑”在没有外部 active-goal snapshot 时也会回落到 report_only。

还有一个当前态漂移：standalone SKILL.md 仍声明 harness-agent-for-goal 与 harness-agents-md-context “remain installed”，但 live skill tree 已不存在这些目录。这说明 public package 与 full stack 的契约一致性不能继续只靠文字维护。

Boundary that must survive

不要把科研或工程执行细节重新塞回 goal-entry。 它应只负责产生更有语义、可验证、可协商的路由结果；阶段图、实验合同、团队和证据门继续由 child owners 承接。

v0.1.0 更新。 2026-07-10 排名前三的 Contract-Conformance Evaluation Harness、Typed Resolver 与 Goal Runtime Profiles 已落地；roadmap approval、milestone acceptance、independent verifier、bounded retry、drift correction、subagent cleanup 与 Claim Firewall 也已进入 declarative contract 和 trace replay。它们现在是基线，不再作为本轮的新建议。

仍未闭合的 seam。 resolver 仍以有序 regex 和硬默认值选择单一 profile；durable state 与 capability manifest 都由 caller 提供；next_owner 只有名字，没有版本协商、接收确认或超时；入口也没有 preview、correction、pause/cancel/amend、risk budget 或重复调用保护。

External Context. 外部主资料显示成熟协议普遍把这些 seam 当成一等合同：MCP lifecycle 在 operation 前协商版本与 capabilities；LangGraph interrupts 用 durable checkpoint 与 thread cursor 恢复；OpenAI Agents SDK handoffs 支持 typed payload、input filter 和 callback；tracing 记录 handoff/guardrail 与 trace identity；IETF Idempotency-Key draft 用稳定 key 与 fingerprint 识别 retry。

当前仓库要求所有 agent-style dispatch 在主线程顺序完成，因此本次 basis verification 由主线程逐项复核，而不是独立 verifier agent；这项降级记录在新 rejection summary 中，不改变证据门槛。

B1. Goal Intent Envelope with abstention

把“我理解成什么”变成可检查、可纠正、可弃权的入口合同。

Description
把 instruction、引用/附件、prior assistant text 与不可信内容分成 provenance lanes；输出候选 intent/profile、negation scope、ambiguity margin、abstention reason 和稳定 correction handle。决策不够明确时先澄清，不允许直接 mutation。
Axis
Intent interpretation and ambiguity
Basis

Downsides
